## 三、哪些样式是 Tailwind class「写死」在 JSX 上

- **`new-feature-chat/page.tsx`**：几乎全部视觉为内联 class（含 `bg-[#0b1020]`、`bg-[#0f172a]`、`left-14` 与侧栏宽度硬耦合、`fixed bottom-0` 底栏等）。
- **`QwenKbAnswerCard.tsx`、`ProcessTimeline.tsx`、`StreamingAnswerDraft.tsx`**：无 CSS Module；颜色、圆角、字号（如 `text-[11px]`、`text-[15px]`）均为工具类拼串。
- **`AppSidebar.tsx`**：`w-14`、`blue-*` / `slate-*` 写死。
- **`kb-update/**` 各 Client/Page**：典型控制台式 `bg-white`、`border-slate-200`、`shadow-sm`、`bg-blue-600` 等，与聊天页深色体系分离。

**未写死在组件里、但可被引用的**：`globals.css` 中的 `--primary`、`bg-background` 等；当前目标路径上**实际使用频率低**（布局 body 仍偏 slate 字面量）。

## 四、适合抽成统一 design tokens 的项

建议优先抽象为 token（CSS 变量或 `@theme` 扩展 + 少量语义组件类），便于一次切换为「现代 AI 产品」中性灰 + 单一强调色风格：

1. **画布层级**：`app-bg`、`chat-panel-bg`、`composer-bg`（替代 `#0b1020` / `#0f172a` 任意值）。
2. **表面与边框**：`surface-elevated`、`border-subtle`、`border-strong`（统一 `slate-700/800` 碎片）。
3. **文本层级**：`text-primary`、`text-secondary`、`text-muted`、`text-accent`（替代多处 `slate-400/500/600`）。
4. **品牌/操作主色**：当前 indigo（聊天）与 blue（kb-update、侧栏）建议收敛为 **primary** + **primary-hover** + **sidebar-active**。
5. **语义状态面**：风险 `amber`、成功 `emerald`、错误 `rose` 可保留色相，但统一 **saturation / alpha** 规则（卡片 ring、border 透明度）。
6. **圆角与阴影**：`rounded-2xl` / `shadow-sm` / `ring-1` 混用，可定义 `radius-chat-bubble`、`shadow-popover`。
7. **字号阶梯**：`text-[10px]`、`text-[11px]`、`text-xs`、`text-sm` 混排，适合 token 化或 Typography 组件。
8. **布局常量**：侧栏 `w-14` 与输入条 `left-14` 应同源（避免改宽漏改底栏）。

## 五、建议分步改造（控制单次改动面）

1. **第一步：统一「壳」与暗色策略**  
   决定全局是「全站 dark」还是「仅聊天路由 dark」。若仅聊天 dark，在 `layout` 或路由 layout 上隔离背景，减少 `body` 与聊天页底色冲突；对齐 `left-*` 与侧栏宽度。

2. **第二步：聊天页 composer + 滚动区**  
   将底部 fixed 输入区改为与设计稿一致的**居中输入条 / 渐变遮罩**（仍可用现有结构），抽 token 替换 hex 与 `slate-*` 碎片。

3. **第三步：消息气泡与头像行**  
   简化为更接近主流产品的**宽内容区 + 弱气泡**或「用户右对齐细条、助手全宽卡片」，减少 assistant 外 `bg-slate-800` 与 `QwenKbAnswerCard` 内多层嵌套视觉重量。

4. **第四步：`QwenKbAnswerCard` 信息架构**  
   合并「结论 / 依据 / 风险 / 建议」视觉层级（留白、标题样式、引用色），统一 `KnowledgeSourcesBlock` 与正文引用样式。

5. **第五步：`ProcessTimeline` 折叠默认态与密度**  
   默认收起或弱化为顶部一条进度，避免首屏「控制台感」过重；与最终答案卡片的间距 token 化。

6. **第六步：`kb-update` 与侧栏**  
   在保持可读前提下，将 `blue-600` 与聊天 `indigo` 对齐；可选引入与聊天一致的顶栏或留白尺度，降低跨页「两个产品」感。

每步完成后做一次**仅视觉回归**（截图对比即可），避免逻辑与样式同 PR。

## 七、与目标（市面 AI 问答页）的差距摘要

- **双主题割裂**：全局浅灰控制台 + 聊天页自定义深色 hex + 侧栏深蓝高亮，与 ChatGPT 类产品常见的**单一中性画布 + 轻分隔 + 一条主色**不一致。  
- **信息密度偏高**：时间线、多色环、ring、多卡片嵌套同时出现，偏「运维面板」而非「对话优先」。  
- **样式不可系统性换肤**：大量任意值与 `slate/indigo/blue` 混用，改品牌色需多处搜索替换。
