## 1. 当前 header 结构

主内容区（侧边栏右侧）为纵向 `flex flex-col`：`flex min-h-0 min-w-0 flex-1 flex-col overflow-hidden`（约 L1293）。

自上而下分为 **三块**（第三块为输入区，严格说不是「标题」但占用底部固定带）：

### 1.1 移动端专用顶栏（`md:hidden`）

- **位置**：约 L1294–L1314。
- **内容**：左侧「历史」、居中标题「对话」、右侧「新对话」。
- **与「Qwen + 阿里云知识库」无关**；这是会话导航条。
- **高度相关 class**：外层 `shrink-0 … px-3 py-2`；内部按钮 `py-2` 等。无与桌面标题行共用的 DOM。

### 1.2 桌面与移动端共用的「产品说明」标题行（你关心的区域）

- **位置**：约 L1316–L1333，外层为 `shrink-0` 的横向 flex 容器。
- **左侧列**（`min-w-0`）：
  - **主标题**：`<h1>` 文案 **「Qwen + 阿里云知识库」**（约 L1323–L1325），class：`text-lg font-semibold tracking-tight`。
  - **副标题**：`<p>` 文案 **「每次提问先检索知识库，再由 Qwen 生成答案」**（约 L1326），class：`mt-1 text-xs text-[var(--app-text-muted)]`。
- **右侧**（条件渲染）：
  - 当 `lastMeta` 非空时，渲染 **模型 / 检索片段** 胶囊：`模型：{lastMeta.model} · 检索片段：{lastMeta.retrievedCount}`（约 L1328–L1331）。
  - `lastMeta` 来自最近一次带 `answerCard` 的助手消息（`deriveLastMetaFromMessages`），在流式回答完成后由 `setLastMeta` 更新（约 L1118）；新会话或无成功卡片时为 `null`，**整块右侧 badge 不显示**。

**该行高度主要由谁控制**：

- 容器：`px-5 py-4 md:px-8`（垂直 padding **py-4** 贡献明显）、`gap-4`、`items-center`。
- 标题：`text-lg`（行高随字体）。
- 副标题：`mt-1` + `text-xs`。
- 无单独 `min-h-*`；总高度由 **padding + 两行文字（或一行 + badge 对齐）** 决定。

### 1.3 消息滚动区与输入区（便于评估「去掉标题后」影响）

- **消息区**：约 L1335–L1413，`min-h-0 flex-1 overflow-y-auto`，宽度与标题行共用 `chatContentMaxClass`（`max-w-[1200px]` 或 `max-w-6xl`）的内层 `mx-auto` 容器（约 L1336）。
- **空状态**：`messages.length === 0` 时在消息卡片内展示一条虚线框提示，文案为 `emptyHint`（`useMemo` 固定为示例问句，约 L986、L1338–L1341）。**与顶部副标题文案无关**。
- **输入区**：约 L1415–L1461，`shrink-0`，`composerMaxClass` 控制最大宽度；与标题行无 DOM 嵌套关系，同属主列 flex 子项。

## 2. 哪些内容可以移除

| 元素 | 建议 |
|------|------|
| `<h1>Qwen + 阿里云知识库</h1>` | **可移除**（或改为极简品牌/留空），与对话内容无数据依赖，移除不破坏请求/渲染逻辑。 |
| 副标题「每次提问先检索知识库…」 | **可移除**；纯说明文案。 |
| 右上角「模型 · 检索片段」胶囊 | **可选移除或迁走**；仅展示态，不参与发送逻辑。移除后 `lastMeta` 状态仍可保留供别处使用，或一并精简状态（属实现细节）。 |

移除 **仅标题行**（保留整行容器为空或极窄）时：消息区为 `flex-1`，标题行 `shrink-0` 变矮后，**剩余纵向空间增加**，消息列表与空状态区域会自然下移边界，**不会**把输入框挤出视口；输入框仍为列末尾 `shrink-0`。

---

## 3. 哪些内容建议保留

- **移动端顶栏**（历史 / 对话 / 新对话）：与 ChatGPT 的「侧栏 + 顶栏入口」类似，**建议保留**，除非产品决定完全依赖侧栏手势（当前侧栏在 `md` 以下隐藏，依赖该顶栏打开历史）。
- **布局骨架**：`shrink-0` 标题行槽位若改为 **单行极简 header**（例如仅保留可选 badge、或空白占位防跳动），可保留与 `chatContentMaxClass` 一致的 **水平对齐与 max-width**，避免消息区与输入区宽度与标题行错位。
- **模型 / 检索信息**（若产品需要可追溯）：**建议以非常驻方式保留**（见第 5 节），而不是硬性要求留在顶部。

## 4. 是否建议把说明文字放到空状态中，而不是顶部常驻

**建议：是。**

理由简要：

- 流水线说明（「先检索再由 Qwen 生成」）对 **首次进入** 或 **零消息** 时最有价值；有消息后重复展示信息密度低、且占用与 ChatGPT 不一致的纵向空间。
- 当前空状态仅有 **示例问句**（`emptyHint`），未解释 RAG 行为；将 **一句短说明** 与示例并列或置于其下，可满足「降低顶部占用」与「仍可发现机制」两全。
- 注意：用户清空会话或「新对话」后仍会看到空状态，说明不会永久丢失。

若希望 **有消息后仍能查看**，可补充：设置菜单、`?` 帮助气泡、或首次回答卡片内一行小字（与 `QwenKbAnswerCard` 的协作属后续实现范围，本次仅建议）。

## 5. 推荐修改方案（设计级，未实施）

以下按侵入性从低到高排列：

1. **最小改动（最接近 ChatGPT）**  
   - 删除或注释掉 L1322–L1327 的 `<h1>` 与 `<p>`。  
   - **可选**：删除 L1328–L1332 的 `lastMeta` 胶囊，或保留在同一行右侧（单行高度）。  
   - 若整行仅剩 badge：可将外层 `py-4` 降为 `py-2`/`py-3`，进一步省高。  
   - 若整行完全不要：可移除 L1316–L1333 整个 `shrink-0` 块；需确认是否仍需要与消息区同宽的「顶部分割线」——当前标题行 **无** `border-b`，分割感主要来自下方消息卡片 `border`；去掉后视觉更「顶到顶」，一般可接受。

2. **推荐（体验完整）**  
   - 去掉顶部常驻说明。  
   - 在空状态（L1338–L1341 附近）增加 **1–2 行** muted 说明 + 保留示例问句。  
   - 「模型 / 检索片段」迁至：最后一次助手 `answerCard` 区域（卡片 meta）、或折叠「本次检索」详情，避免顶栏占位。

3. **与移动端统一**  
   - 桌面端若也希望有轻量标题，可用 **「对话」** 或当前会话标题（需从 `sessions`/`activeSessionId` 取标题 state），与移动端文案对齐；仍比「Qwen + 阿里云知识库」更贴近通用聊天产品。

## 6. 移除标题后对消息区高度、输入框、空状态的影响（结论）

| 项目 | 影响 |
|------|------|
| 消息区高度 | **增加**。标题行为 `shrink-0`，高度减小后 `flex-1` 消息滚动区占用更多视口。 |
| 输入框位置 | **仍在列底部**（`shrink-0`），相对视口 **不被顶走**；仅其上方可滚动区域变高。 |
| 空状态 | **逻辑不变**；仍仅在 `messages.length === 0` 时出现。若把说明迁入空状态，空状态 **内容更丰富**，与 header 解耦。 |
