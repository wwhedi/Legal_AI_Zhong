## 1. 当前代码状态总结

### 1.1 `Legal_AI/config/legal_prompts.py`（RAG 最终回答）

- **结构形态**：系统提示「五、回答结构」要求**按固定顺序**输出；其中明确 **第 1、2、3、5 节为强制**；**第 4 节「可执行操作步骤」仅适合时输出**（不适配则整节省略，不用占位句）。用户提示再次强调第 4 节可选、**其余四节不得省略**。
- **标题命名**：正式约束使用 **「1) 一句话结论」「2) 你现在最该做」「3) 需要注意」「4) 可执行操作步骤」「5) 法律依据」**；**未**使用「关键判断标准」「核心判断要点」等独立小节标题（文中仅有「关键法律判断」等**引用/写作要求**，非模块标题）。
- **“全部模块每次必出”问题**：**是**——除第 4 节外，1/2/3/5 在 prompt 中均为**强制**，与「仅按需输出模块池」目标不一致。
- **“你现在可以这样处理”**：当前规范为 **「2) 你现在最该做」**；**未**出现目标命名「你现在可以这样处理」。
- **第 2 节条数上限**：**已**要求 **2～3 条、不得超过 3 条**。
- **“当前知识库未提供……”在「需要注意」中的位置**：**已**要求：一般注意点在前；**依据不足/知识库未覆盖**类须**单独最后一条**，且**整句以「当前知识库未提供」开头**；并禁止把该信息只放在句末。
- **维权/流程/申请/起诉等是否必须出操作步骤表**：**未**对「索赔/赔偿主张/起诉/仲裁」等作**逐字强制**；表述为 **流程/申请/办理/维权步骤类**且**有按顺序指导必要**时输出表格，并举例「劳动仲裁怎么申请、支付令、**起诉流程**、工伤认定…」等；**判断类/要件类原则上省略**。
- **禁止编造案例号、判决号、期限、赔偿标准**：
  - **期限/天数**：**有**——表格中不得编造具体天数，无依据写「未提供明确时限」等。
  - **法条/元数据**：**有**——不得编造法规名、条文编号、链接、日期、时效等；字段缺失写「未提供」。
  - **案例编号/判决号/赔偿标准**：RAG 最终回答主 prompt **未单独逐条**写「不得编造案例号/案号/判决号/赔偿标准」；**未**以专条形式覆盖判例引用。用户提示中的示例句涉及「违约金标准」属说明语气，**非**整段「禁止赔偿标准造假」的独立规则（与目标中的显式禁止相比略弱）。

### 1.2 `Legal_AI/frontend/src/app/new-feature-chat/page.tsx`（`normalizeAnswer`）

- **支持的“标题/小节”类型**（`SectionKind`）：`conclusion` | `basis` | `risks` | `actionAdvice` | `actionSteps`；由 `SECTION_HEADER_RULES` 按行匹配切换当前桶。
- **“影响结果的关键事实”**：**未**在规则表中出现，**无**独立桶位。
- **“你现在可以这样处理”**：**未**匹配；第二节兼容 **「你现在最该做」「现在最该做」** 及「行动建议」「下一步」「你可以这样做」「建议」等。
- **“特别说明 / 需要注意”**：**有**「需要注意」及多种风险相关别名；**未**见「特别说明」专用规则。
- **标题是否残留进正文**：多数规则通过 `stripSectionHeaderLine` / `flexStrips` 剥离；另有 `peelFollowingSection` 处理**标题与正文挤在同一行**的情况。**若模型使用未注册的新标题**，仍可能落入当前 `mode` 桶或结论段，存在残留风险。
- **「需要注意」是否可能重复**：解析侧依赖标题行切换桶；展示侧 `QwenKbAnswerCard` 另有剔除重复标题行的逻辑（见下节）。**跨段落语义重复**仍可能发生。
- **旧标题兼容**：结论段兼容「一句话结论」「简短结论」「核心结论」「直接回答」「结论」等；依据段兼容「法律依据」「引用依据」「相关依据」等；风险与行动建议也有多种别名。

### 1.3 `Legal_AI/frontend/src/components/chat/QwenKbAnswerCard.tsx`

- **外框/卡片感**：根节点为 **`rounded-2xl` + `border` + `shadow-[var(--app-shadow-sm)]` + `bg-white` + `p-5`**，**明显卡片化**。
- **模块顺序**（自上而下）：**问题/模型元信息** → **结论** → **你现在最该做**（始终有区块；占位时 muted） → **需要注意**（有有效条目才渲染） → **可执行操作步骤**（经 `shouldShowActionStepsSection` 过滤后） → **引用依据**（有实质内容时可折叠展开） → **知识库来源**（默认可折叠区域）。
- **「关键判断标准」「核心判断要点」**：UI **未**使用这两类展示标题。
- **「你现在最该做 / 你现在可以这样处理」列表形态**：`ActionChecklistBlock` 使用 **`list-none` + 左侧序号 `1.` `2.` …**，为**编号列表**。
- **「需要注意」与「当前知识库未提供…」顺序**：
  - **多条之间**：`sortRiskBlocksBoundaryLast` 将含「当前知识库未提供」等**边界类短语**的条目**排到列表最后**，与 prompt「最后一条」在**多条场景**下一致。
  - **单条内部**：`promoteKnowledgeBoundaryClause` 会把「当前知识库未提供…」等**挪到该条叙述的前面**，与「必须以当前知识库未提供开头」一致；若用户期望「整段叙述末尾才出现边界句」，则与单条内前置策略不同（当前实现偏「句首突出」）。
- **大面积色块/强边框**：主卡片为白底+边框+阴影；内部多为分隔线与弱边框；无大面积纯色 Hero 块，但**整体仍为强卡片容器**。
- **`[n]` 引用悬浮**：`renderTextWithCitations` + `InlineCitationMark` + `CitationPopover` **保留**（点击/悬停展示）。

### 1.4 `Legal_AI/frontend/src/components/chat/ActionStepsTable.tsx`

- **旧表头 `| 步骤 | 操作内容 | 时间/时限 |`**：**支持**。`headerMatchesRequirements` 要求表头含「步骤」、操作列（「操作内容」或单列「操作」）、第三列与时间/时限相关（「法律时限」「时限」「时间」）。
- **新表头 `| 阶段 | 应对措施 | 法律要点 |`**：**不支持**——缺少「步骤」、第三列也不满足现有时间/时限识别，**解析返回 `null`**。
- **Emoji**：组件内**无** emoji 逻辑。
- **`[n]` 悬浮**：通过父组件传入的 `renderCell`（通常为 `renderTextWithCitations`）**保留**。
- **解析失败回退**：`QwenKbAnswerCard` 在 `parseActionStepsTable` 为 `null` 时，对含 `|` 的原文走 **`prewrap` 纯文本**；否则走**列表**或整段文本，**不会空白丢失**。

### 1.5 `Legal_AI/frontend/src/components/chat/ProcessTimeline.tsx`

- **`defaultOpen`**：`useState(defaultOpen ?? true)` —— **未传时默认展开**。
- **`new-feature-chat/page.tsx` 用法**：已落库消息上 **`ProcessTimeline` → `defaultOpen={false}`**（完成后默认收起）；流式加载中 **`defaultOpen={true}`**（生成中默认展开）。
- **视觉焦点**：生成流水线进行时，时间线**位于回答卡片上方**且默认展开，仍会占用纵向空间；完成后收起可降低干扰，但外层聊天区仍有 **大容器 border + 阴影**（`page.tsx` 中消息列表外包一层），整体「流程感」仍在。
